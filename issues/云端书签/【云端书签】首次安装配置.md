# 01 首次安装配置教程  
  
本文档介绍如何安装云端书签插件并完成首次 WebDAV 配置。  
  
---  
  
## 安装扩展  
> Tips: 目前已经可以在浏览器插件商店进行下载  
### Chrome / Edge 浏览器  
  
1. 打开浏览器，在地址栏输入：  
   - Chrome: `chrome://extensions/`  
   - Edge: `edge://extensions/`  
  
2. 在右上角开启 **开发者模式**。  
  
3. 点击 **加载已解压的扩展程序**。  
  
4. 选择 `cloud_bookmark` 项目根目录。  
  
5. 安装成功后，浏览器工具栏会出现「云端书签」图标。  
  
### Firefox 浏览器  
  
1. 打开 `about:debugging#/runtime/this-firefox`。  
  
2. 点击 **临时载入附加组件**。  
  
3. 选择项目根目录下的 `manifest.json`。  
  
4. 安装成功后，工具栏会出现插件图标。  
  
> 注意：临时载入的扩展在浏览器重启后会失效，正式使用请打包为 `.xpi` 安装。

## 首次配置 WebDAV  
  
### 打开设置页面  
  
有两种方式进入设置：  
  
- **方式一**：点击工具栏插件图标 → 在弹窗右上角点击「设置」图标。  
- **方式二**：在扩展管理页面 → 找到「云端书签」→ 点击「扩展选项」。  
  
### 填写配置信息  
  
在「云端同步配置」区域填写以下信息：  
  
| 配置项 | 说明 | 示例 |  
|--------|------|------|  
| WebDAV 地址 | 你的 WebDAV 服务地址 | `https://dav.jianguoyun.com/dav/`（坚果云）<br>`https://example.com/remote.php/dav/files/username/`（Nextcloud） |  
| 用户名 | WebDAV 用户名 | 一般是账号或邮箱 |  
| 密码 / Token | WebDAV 密码或应用专用密码 | 部分服务需要创建应用专用密码 |  
| 同步路径 | 插件在 WebDAV 上的工作目录 | 默认: `/bookmarks/` |  
| 同步间隔 | 后台自动同步间隔（分钟） | 默认: `5` |  
  
### 测试并保存  
  
1. 填写完成后点击 **测试连接**。  
   - 成功：提示「连接成功」，可以继续保存。  
   - 失败：根据错误提示检查地址、用户名、密码。  
  
2. 点击 **保存配置**。  
   - 插件会保存配置到本地。  
   - 自动从云端拉取设置。  
   - 注册当前设备。  
   - 执行首次同步。

## WebDAV 配置迁移  
  
如果你已经在其他设备上配置好了，可以快速迁移配置：  
  
### 从另一台设备导出  
  
1. 在已配置好的设备上打开设置页。  
2. 在「WebDAV 配置工具」区域点击 **导出到剪贴板**。  
3. 配置会以三行格式复制到剪贴板：  
   ```  
   webdav地址  
   webdav用户  
   webdav密码  
   ```  
### 在新设备导入  
  
1. 在新设备设置页点击 **导入配置**。  
2. 在弹出框中粘贴三行格式的配置。  
3. 点击「导入」，表单会自动填充。  
4. 检查无误后点击「保存配置」完成。

## 常见问题  
  
### Q: 提示「无法连接到 WebDAV 服务器」  
A: 检查以下几点：  
- 地址是否正确（必须包含完整路径，如 `/dav/` 结尾）  
- 服务端是否开启 HTTPS  
- 用户名和密码是否正确  
- 网络是否能访问该地址  
  
### Q: 保存配置后本地原有书签会丢失吗？  
A: 不会。首次保存配置检测到本地已有书签时，会自动归档合并到云端，保证不会覆盖丢失。  
  
### Q: 支持哪些 WebDAV 服务？  
A: 支持所有标准 WebDAV 协议服务，包括：  
- 坚果云  
- iCloud WebDAV  
- Nextcloud  
- ownCloud  
- 阿里云盘 WebDAV  
- 群晖 WebDAV  
- 其他任何支持标准 WebDAV 的服务
